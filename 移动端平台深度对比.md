## 一、统一移动端路线图

> 本文件合并了"移动端适配方案"与"移动端平台深度对比"，统一为四阶段路线。

| 阶段 | 方案 | 覆盖平台 | 成本 | 时间 | 触发条件 |
|------|------|---------|------|------|---------|
| **Phase 1** | PWA（响应式 CSS） | 全部 | ¥0 | 1-2 周 | 现在 |
| **Phase 2** | 极简移动版（搜索+评级+推送） | 全部 | ¥0 | 2-3 天 | 有 200+ 用户反馈后 |
| **Phase 3** | Uni-app 全平台 | Android + iOS + 鸿蒙 + 小程序 | ¥0 | 2-3 周 | 用户要求原生体验时 |
| **Phase 4** | 鸿蒙平板原生 | 鸿蒙平板 | ¥2,000 | 2-3 周 | 有收入 + 华为市场认证后 |

### 各平台单独优先级

| 平台 | 原生优先级 | 理由 |
|------|-----------|------|
| **PWA（通用）** | 🥇 P0 | 覆盖所有，零成本，立即做 |
| **Uni-app 全平台** | 🥈 P1 | Vue 语法复用，一套代码出所有平台 |
| **HarmonyOS** | 🥉 P2 | 蓝海窗口 + 平板论文场景，但需 Uni-app 先打底 |
| **Android** | 4 P3 | Uni-app 覆盖，无需独立原生 |
| **iOS** | 5 P3 | 成本最高、覆盖率最低，有收入后再考虑 |

## 二、各平台逐项拆解

### 2.1 Android

| 维度 | 详情 |
|------|------|
| **开发语言** | Kotlin（推荐）/ Java（旧） |
| **开发工具** | Android Studio（免费） |
| **模拟器** | 内置 AVD，免费 |
| **测试设备** | 一部中端机 ¥1500（足够） |
| **上架平台** | Google Play / 华为应用市场 / 小米应用商店 等 |
| **上架费用** | Google Play $25 一次性 / 国内商店多数免费 |
| **审核周期** | Google Play 几小时 / 国内 1-3 天 |
| **包体大小** | APK ~10-50MB |
| **用户覆盖** | 国内 ~75%，全球 ~70% |

#### 开发流程

```
1. 环境搭建              → Android Studio + JDK（半天）
2. 基础框架              → Activity → Fragment → RecyclerView（2天）
3. API 对接              → Retrofit + OkHttp（1天）
4. 论文搜索页面           → SearchView + 结果列表（1天）
5. 评级详情页             → 雷达图（MPAndroidChart）+ 卡片（2天）
6. 推送通知              → FCM / 厂商推送（1天）
7. 测试 + 上架            → 真机测试 + 商店审核（2天）

总开发时间：3-4 周（独立原生）
如果用 Flutter：2-3 周（Android + iOS 双端）
```

#### 技术难点

| 难点 | 说明 | 解决方案 |

| 权限管理 | 存储/通知权限各版本不同 | 运行时权限请求 |

### 2.2 iOS

| 维度 | 详情 |
|------|------|
| **开发语言** | Swift + SwiftUI（推荐）/ Objective-C（旧） |
| **开发工具** | Xcode（免费） |
| **模拟器** | 内置 Simulator，免费且强大 |
| **测试设备** | 真机测试必须（模拟器不覆盖全部） |
| **上架平台** | 仅 App Store |
| **上架费用** | **$99/年** 苹果开发者计划（必须） |
| **审核周期** | 1-7 天（严格，可能被拒多次） |
| **包体大小** | IPA ~20-80MB |
| **用户覆盖** | 国内 ~25%，全球 ~30% |

#### 开发流程

```
1. 环境搭建              → Xcode + 开发者账号（半天）
2. 基础框架              → SwiftUI NavigationStack → List（2天）
3. API 对接              → URLSession + Codable（1天）
4. 论文搜索页面           → .searchable + List（1天）
5. 评级详情页             → Swift Charts（雷达图）+ Card（2天）
6. 推送通知              → APNs（1天）
7. 适配刘海屏/灵动岛       → SafeArea + GeometryReader（1天）
8. 测试 + 审核            → TestFlight + App Store Connect（3天）

总开发时间：4-6 周（独立原生）
```

#### 成本明细

| 项目 | 一次性 | 年费 | 说明 |

| Mac 电脑 | ¥0（云 Mac ¥200-400/月） | ¥2,400-4,800/年 | 可以不用买：云服务按需付费，有收入后再投资硬件 |
| Xcode | ¥0 | — | 免费 |
| 测试设备 | ¥5,000+ | — | 1 部 iPhone |
| Apple Developer | — | **$99** | **必须每年续费** |
| **总一次性** | **≈ ¥5,000（云Mac方案）** | **¥688/年** | 云 Mac ¥200-400/月 + iPhone 测试机 ¥5,000 |

#### 技术难点

| 难点 | 说明 | 解决方案 |
|------|------|---------|
| Mac 硬性要求 | 传统方式必须 | 方案A：Mac云服务（MacStadium/MacinCloud）¥200-400/月；方案B：买 Mac mini ¥4000；方案C：React Native/Flutter + CI/CD 云打包（不需本地Mac） |

| SwiftUI 兼容性 | iOS 15 / 16 / 17 行为不同 | 设最低版本 iOS 16 |

### 2.3 HarmonyOS（鸿蒙）

| 维度 | 详情 |
|------|------|
| **开发语言** | ArkTS（推荐）/ Java / JS |
| **开发工具** | DevEco Studio（免费，Windows/Mac） |
| **模拟器** | 内置，需华为账号登录 |
| **测试设备** | 中端华为手机 ¥2000 |
| **上架平台** | 华为应用市场（AppGallery） |
| **上架费用** | **免费**（个人/企业开发者均免费） |
| **审核周期** | 1-3 天（比 iOS 宽松） |
| **包体大小** | HAP ~5-30MB |
| **用户覆盖** | 国内鸿蒙设备 ~8 亿，纯 HarmonyOS 5+ 占比增长中 |

#### 为什么 HarmonyOS 值得单列

```
鸿蒙的特殊价值（对论文工具而言）：
  1. 国内高校信息化采购偏向华为生态（平板 + 笔记本 + 手机全家桶）
  2. 华为平板国内教育市场份额第一（学生看论文场景）
  3. 华为应用市场对学术类 App 有专区曝光
  4. 鸿蒙 NEXT 原生优先，原生 ArkTS 应用获应用市场流量倾斜（非"完全不兼容"，而是"原生体验更优 + 市场曝光更高"）
     → 早期进入者少，蓝海窗口
  5. 开发生态年轻，开发者少 → 精品 App 稀缺 → 更容易被推荐
```

#### 开发流程

```
1. 环境搭建              → DevEco Studio + 华为开发者账号（半天）
2. 基础框架              → ArkUI Navigation → List（2天）
3. API 对接              → @ohos.net.http（1天）
4. 论文搜索页面           → Search + ListItem（1天）
5. 评级详情页             → Canvas 自定义雷达图 + 卡片（2天）
6. 推送通知              → 华为 Push Kit（0.5天）
7. 平板适配              → 横屏布局（1天）← 鸿蒙平板是杀手场景
8. 测试 + 上架            → 真机 + 应用市场（2天）

总开发时间：2-3 周（ArkTS 学习曲线低，类 TypeScript）
```

|------|--------|------|------|
| 开发设备 | ¥0 | — | DevEco 支持 Windows |
| DevEco Studio | ¥0 | — | 免费 |
| 测试设备 | ¥2,000 | — | 华为手机/平板 |
| 华为开发者账号 | ¥0 | — | 个人免费注册 |

#### 技术难点

| 难点 | 说明 | 解决方案 |
|------|------|---------|

## 四、跨平台方案（一石三鸟）

如果嫌三个平台都开发太累：

| 方案 | 覆盖 | 学习成本 | 开发时间 | 适合吗 |
|------|------|---------|---------|--------|
| **Uni-app** | Android + iOS + 鸿蒙 + 小程序 | 低（Vue 语法） | 2-3 周全平台 | ✅ 最适合 |
| Flutter | Android + iOS | 中（Dart） | 3-4 周双端 | 🟡 |
| React Native | Android + iOS | 中 | 3-4 周 | 🟡 |

> **Uni-app 对本项目是最优跨平台方案**：你已经在用 Vue 3，Uni-app 就是 Vue 语法，直接把现有组件迁移过去。一套代码 → Android APK + iOS IPA + 鸿蒙 HAP + 微信小程序。

### Uni-app 具体操作

```
1. 安装 HBuilder X（免费）
2. 创建 uni-app 项目（选 Vue 3 模板）
3. 把现有 Vue 组件拷贝过去（80% 代码复用）
4. 适配条件编译（平台差异只需少量 ifdef）
5. 一键打包：Android / iOS / 鸿蒙 / 小程序

总耗时：2-3 周（基于已有 Vue 代码）
AI 参与度：85%（组件迁移 + 条件编译 + 打包配置）
成本：¥0（HBuilder X 免费，个人开发者免费打包）
```

## 六、鸿蒙平板的特殊机会

这是一个被大多数人忽略的点：

```
鸿蒙平板 + 论文工具：
  学生用平板看 PDF 论文（比手机大，比电脑轻）
    ↓
  在平板上搜索 → 看评级 → 决定要不要精读
    ↓
  平板版本的论文评级工具 = 刚需但没人做的场景
    ↓
  华为应用市场"教育/学术"类目竞争极低
```

**如果只做一个原生平台，做鸿蒙平板版。** 原因：场景完美匹配、竞争蓝海、华为市场有流量倾斜。

## 🎯 最终建议

| 阶段 | 做什么 | 花多少 |
|------|--------|--------|
| **立即** | PWA 响应式（已有 Vue 项目直接改 CSS） | ¥0，1 周 |
| **有用户后** | Uni-app 全平台（Android + iOS + 鸿蒙 + 小程序） | ¥0，2-3 周 |
| **有收入后** | 鸿蒙平板原生（平板看论文体验优化） | ¥2,000，2-3 周 |
| **规模大了** | iOS 原生（如果要进 App Store 精品推荐） | ¥13,000 + ¥688/年 |

**一句话**：先用 PWA 让所有手机都能用 → 再用 Uni-app 一套代码出所有平台 → 最后看用户数据决定深耕哪个平台。
