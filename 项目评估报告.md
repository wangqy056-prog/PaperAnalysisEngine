## 四、架构质量评估

### ✅ 优势

1. **前后端分离彻底**：FastAPI ↔ Vue 3 独立部署，前端 Vite 代理 /api
2. **API 文档自动生成**：访问 /docs 即 Swagger UI
3. **LLM 三模型降级链**：任一模型故障，自动切换下一个
4. **模块化清晰**：每个 .py 单一职责，易于维护
5. **技术栈迁移正确**：Streamlit（原型）→ FastAPI+Vue（生产），解决了 DOM 残留顽疾
6. **反反爬机制**：Playwright headless Chromium 突破 React SPA 的 SSR 缓存
7. **数据去重体系**：标题查重 + paper_id 去重 + 评级 DELLETE-before-INSERT

### ⚠️ 技术债务

| ID | 严重度 | 问题 | 位置 | 商业化影响 |
|----|--------|------|------|-----------|
| **T1** | 🔴 致命 | **API Key 硬编码** | `llm_analyzer.py:22-34` | 代码泄露=密钥泄露，Enterprise客户绝不会接受 |
| **T2** | 🔴 高 | **两套评级引擎共存** | `engine/rating.py` vs `backend/rating_engine.py` | 功能重叠，但暂时不影响用户使用 |
| **T3** | 🔴 致命 | **缺 Docker 化** | 根目录 | Enterprise 客户无法一键部署 → 直接拒绝合作 |
| **T4** | 🔴 致命 | **缺测试覆盖** | 全局 | 迭代时出 Bug 摧毁用户信任；仅 1 个骨架 test |
| **T5** | 🟡 中 | **二进制文件入库** | `backend/papers.db`(2MB)、`data/papers.db`(88KB) | 仓库膨胀，不影响功能 |
| **T6** | 🟢 低 | **CLI 入口重叠** | `main.py` vs `backend/cli.py` | 不影响 |
| **T7** | 🟢 低 | **端口硬编码** | `api/main.py` + `vite.config.js` | 不影响，Docker 化后自动消失 |

> **T1/T3/T4 必须在任何商业化动作之前解决——它们是 Enterprise 客户合作的准入门槛，不是可选优化。**
