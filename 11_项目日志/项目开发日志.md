### 2026-06-28 — 修复 Vue Router 导航卡死（问题 #17）

**问题现象：**
用户反馈"开始可以切换但是点到论文推荐还是哪个功能之后就不能切了"——进入「论文推荐」页面后所有导航失效。

**根因：**
1. 后端 [recommendation_engine.py](file:///d:/Trae/chat/Paper_Analysis_Engine/backend/recommendation_engine.py) 的 `recommend_by_*` 函数直接从数据库 `SELECT p.authors`，返回的是 JSON 字符串而非数组
2. 前端 [Recommendations.vue:57](file:///d:/Trae/chat/Paper_Analysis_Engine/frontend/src/views/Recommendations.vue#L57) `(rec.authors || []).slice(0, 3).join(', ')` 假设 authors 是数组；对字符串 `.slice(0,3)` 返回子串，子串无 `.join` 方法 → TypeError
3. 渲染错误冒泡到 `<RouterView>`，触发 Vue 内部 vnode 状态损坏（`Cannot destructure property 'type' of 'vnode' as it is null`），后续路由切换全部失效

**修复：**
- 后端：`recommendation_engine.py` 新增 `_normalize_paper()`，在 `get_recommendations` 入口统一解析 JSON 字段为数组
- 后端：`db.py:search_papers` 添加相同归一化（影响 Search/Leaderboard）
- 前端：`Recommendations.vue` 新增 `authorsList()` 防御性辅助函数（双保险）

**验证：**
用 Chrome DevTools Protocol 模拟点击 `/` → `/recommendations` → `/leaderboard` → `/`，`allSwitched: true`，`__errors: []` ✅

**详细方案：** 见 [问题解决方案档案.md](./问题解决方案档案.md) #17

### 2026-06-28 — 修复知识图谱加载失败 400（问题 #18）

**问题现象：**

- [api/main.py:type_map](file:///d:/Trae/chat/Paper_Analysis_Engine/backend/api/main.py#L311-L333) 兼容三种命名（简短/长形式/中文）
- [api/main.py:edges](file:///d:/Trae/chat/Paper_Analysis_Engine/backend/api/main.py#L346-L356) 同时输出 `source/target` + `from/to`

**附带修复：** 8000 端口僵尸 socket 占用，临时切换后端到 8001，[vite.config.js](file:///d:/Trae/chat/Paper_Analysis_Engine/frontend/vite.config.js) 代理同步更新

**验证：** API 返回 200 + 正确的 nodes/edges，edges 含 `from/to/source/target` ✅

```python
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt
import numpy as np
from io import BytesIO

def generate_scorecard(paper, rating) -> bytes:
    fig, ax = plt.subplots(figsize=(6, 6), subplot_kw=dict(polar=True))
    labels = ['Academic\nImpact', 'Commercial\nPotential', 'Innovation\nIndex',
              'Reproducibility', 'Combo\nValue']
    values = [rating['academic_impact'], rating['commercial_potential'],
              rating['innovation_index'], rating['reproducibility'],
              rating.get('combo_value', 50)]
    values += values[:1]
    angles = np.linspace(0, 2*np.pi, len(labels), endpoint=False).tolist()
    angles += angles[:1]
    
    ax.fill(angles, values, alpha=0.25, color='#e94560')
    ax.plot(angles, values, linewidth=2, color='#e94560')
    ax.set_xticks(angles[:-1])
    ax.set_xticklabels(labels, fontsize=10)
    ax.set_ylim(0, 100)
    plt.title(f"PAE Rating: {rating['grade']}\n{paper['title'][:60]}",
              fontsize=12, fontweight='bold', pad=20)
    fig.text(0.5, 0.02, 'Generated by PAE Rating | pae.example.com',
             ha='center', fontsize=8, color='grey', alpha=0.7)
    
    buf = BytesIO()
    plt.savefig(buf, format='png', dpi=150, bbox_inches='tight',
                facecolor='white', edgecolor='none')
    plt.close()
    buf.seek(0)
    return buf.read()
```

API 端点（`backend/api/main.py`）：

```python
from fastapi.responses import Response
from scorecard_generator import generate_scorecard

@app.get("/api/paper/{paper_id}/scorecard")
async def get_scorecard(paper_id: str):
    paper = get_paper(paper_id)
    rating = get_paper_rating(paper_id)
    if not paper or not rating:
        raise HTTPException(404)
    img_bytes = generate_scorecard(paper, rating)
    return Response(content=img_bytes, media_type="image/png")
```

### 2026-06-30 — 修复批量导入主题库显示为空（问题 #28）

**问题现象：**
用户反馈 [BatchImport.vue](file:///d:/Trae/chat/Paper_Analysis_Engine/frontend/src/views/BatchImport.vue) 的「多主题导入」页面主题库显示为空，无法选择分类和主题。

**根因（API 数据格式契约不一致）：**
- 后端 [api/main.py](file:///d:/Trae/chat/Paper_Analysis_Engine/backend/api/main.py) `/api/batch-import/topics` 直接 `return TOPIC_LIBRARY`
- [batch_import_topics.py](file:///d:/Trae/chat/Paper_Analysis_Engine/backend/batch_import_topics.py) 的 `TOPIC_LIBRARY` 是字典：`{"人工智能": [...], "量子计算": [...], ...}`（10 个分类，41 个主题）
- 前端 `loadTopics()` 用 `Array.isArray(data)` 检查，对字典返回 false → fallback 到 `[]` 空数组

**修复（[BatchImport.vue](file:///d:/Trae/chat/Paper_Analysis_Engine/frontend/src/views/BatchImport.vue#L173) `loadTopics()`）：**
兼容数组和字典两种返回格式：
```javascript
if (Array.isArray(data)) {
  arr = data  // 后端返回 [{category, topics}, ...]
} else if (data && typeof data === 'object') {
  arr = Object.entries(data).map(([category, topics]) => ({ category, topics }))
} else {
  arr = []
}
```

**问题 2 排查结果（无 bug）：**

| 文件 | 检查项 | 结论 |
|------|--------|------|
| [KnowledgeGraph.vue](file:///d:/Trae/chat/Paper_Analysis_Engine/frontend/src/views/KnowledgeGraph.vue#L86-L87) | `data.nodes \|\| []` + `data.edges \|\| []` | ✅ 与后端返回的 `{nodes, edges}` 对象格式匹配 |
| [AIAnalysis.vue](file:///d:/Trae/chat/Paper_Analysis_Engine/frontend/src/views/AIAnalysis.vue#L158) | 防御性检查 authors 字段 | ✅ 无字典/数组解析问题 |
| Dashboard.vue | 仪表盘统计 | ✅ 直接读字段，无解析问题 |

**验证：**
- API `/api/batch-import/topics` 返回 dict，10 个分类，41 个主题 ✅
- 前端刷新后应显示完整主题库（不需要重启服务）✅

**经验教训：**
后端返回字典 vs 数组是常见的数据格式契约问题。前端解析时应同时兼容两种格式，或后端统一返回数组。后续接口约定：**列表型数据统一返回数组**，如需分类信息用对象数组 `[{category, items}]` 而非字典 `{"category": items}`。
