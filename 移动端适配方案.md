## 一、核心结论

**移动端不是禁选项，是低优先级项。**
- 核心用户（研究者）用电脑看论文
- 先用桌面版抓住用户
- 等用户开始抱怨"手机上查不了评级"时再做

---

## 二、方案对比

| 方案 | 工作量 | 体验 | 适配成本 | 推荐 |
|------|--------|------|---------|------|
| **A. 响应式 Web** | 1-2 周 | ⭐⭐⭐ | 改 CSS + 组件适配 | 🟡 |
| **B. PWA（可安装）** | 在 A 基础上 +3 天 | ⭐⭐⭐⭐ | 加 manifest + Service Worker | ✅ 推荐 |
| **C. 小程序** | 3-4 周 | ⭐⭐⭐⭐ | 独立前端代码 | ❌ 太重 |
| **D. Flutter/RN 原生** | 8-12 周 | ⭐⭐⭐⭐⭐ | 完全独立开发 | ❌ 太重 |

> **推荐 A+B 组合：同一套 Vue 代码适配手机 + PWA 可安装到桌面。零额外维护成本。**

## 三、具体改造项

```
当前问题 → 修改方案

导航栏太长        → 底部 Tab 切换（移动端）
表格太宽          → 卡片列表替代（论文列表）
雷达图太大        → 响应式缩放（max-width: 100%）
操作按钮密集      → 长按菜单 / 滑动操作
图片加载慢        → 懒加载 + WebP
搜索输入          → 顶部固定搜索栏
侧边栏            → 折叠为汉堡菜单
```

### 关键 CSS 断点

```css
/* 手机 */
@media (max-width: 768px) { ... }

/* 平板 */
@media (max-width: 1024px) { ... }
```

## 五、极简移动版（快速方案）

如果不想做完整适配，只做一个极简版：

```
手机端只做 3 件事：
  1. 搜索论文（输入框 → 列表）
  2. 看评级（五维雷达图 + 等级）
  3. 推送通知（新论文提醒）

不做的事：
  ✗ 知识图谱（屏幕太小）
  ✗ 批量导入（电脑上做）
  ✗ 详细分析（电脑上做）
  ✗ 收藏夹管理（电脑上做）
```

耗时：2-3 天。

## 六、实施时机

| 时间点 | 动作 | 理由 |
|--------|------|------|
| **现在** | 不改 | 核心用户用电脑 |
| **Phase 2 完成后**（3-4 周） | 花 1 天加关键 CSS | 让页面在手机上不崩 |
| **Phase 3 完成后**（5-6 周） | 2-3 天极简移动版 | 搜索 + 评级 + 推送 |
| **有 500 用户后** | 评估是否需要 PWA 完整版 | 用户反馈驱动 |
